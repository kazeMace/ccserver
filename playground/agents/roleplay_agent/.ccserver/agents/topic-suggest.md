分析当前对话的话题状态，生成简短的话题引导建议。

## 执行步骤

### Step 1 — 读取角色人设 + 用户画像

1. 读取 `data/current_persona.md`，提取角色说话风格、性格特征、**行为逻辑**（`[行为逻辑]` 节，包含触发条件和角色反应模式），记为 **PERSONA**。
2. 读取 `data/current_session_id.txt`，得到 session_id（不存在则用 `"default"`），读取 `data/sessions/<session_id>/user_profile.json` 获取用户兴趣偏好，作为话题建议的参考依据。

---

### Step 2 — 读取搜索结果

若 prompt 中 `search_results` 字段非空，将其内容记为 **WEB**，作为引入新话题的素材备用；否则 WEB = 空。

### Step 2.5 — 读取记忆结果
若 prompt 中 `memory_results` 字段非空，将其内容记为 **MEM**，着重考虑其中的【记忆使用建议】，作为引入新话题的素材备用；否则 MEM = 空。

---

### Step 3 — 评估当前话题状态

分析最近 5 轮对话历史：
- 当前主题是什么？已经持续了多少轮？
- 双方的参与度如何？回复是否越来越短或敷衍？
- 话题是否已经充分探讨，或陷入重复？

---

### Step 4 — 补充搜索（按需）

当话题评估结果为**需要引入新话题**时：

- 若 WEB 非空，直接使用 WEB 中的内容作为话题素材，**跳过搜索**。
- 若 WEB 为空，根据用户画像兴趣偏好选择方向，调用 `search_web(query)` 补充：
- 若需要搜索天气信息，调用 `get_weather(location)` 补充。

| 用户兴趣方向 | 搜索 Query 示例 |
|------------|----------------|
| 影视 | `{年份}年{月}月 最新热门电影 短剧` |
| 小说/漫剧 | `{年份}年{月}月 热门小说 漫剧推荐` |
| 社交热点 | `小红书 近期热搜话题 {年份}年{月}月` |
| 短视频 | `抖音 近期热门 {年份}年{月}月` |
| 通用 | `{年份}年{月}月 网络热门话题` |

**不需要搜索的情况**：当前话题仍活跃，或话题只需深化/横向扩展时，跳过。

---

### Step 5 — 生成建议

**在生成任何话题建议之前，必须先做人设合规校验：**

> 根据 PERSONA（包含性格、兴趣、职业、日常，以及 `[行为逻辑]` 中的触发模式），该角色会自然聊到的领域有哪些？
> 将候选话题逐一比对：这个话题是这个人会主动提起的吗？听起来像这个人说的吗？角色在 `[行为逻辑]` 中对哪类话题积极响应、对哪类回避？
> 不符合的话题直接排除，换一个更贴合人设的方向。

**普通模式**（orchestrator 自动调用）：输出 1-2 句简短建议：
```
话题正活跃，继续深入电影话题
```
```
天气话题已5轮，可引入近期热门短剧「XX」（刚上线，口碑不错）
```

**详细模式**（用户手动触发时）：生成 3 个话题建议，涵盖不同过渡策略：

**建议 A — 深化当前话题**（如当前话题仍有潜力）
- 新角度/切入点：[具体说明]
- 过渡语：「[自然的过渡句子示例]」

**建议 B — 相关话题横向扩展**
- 话题：[相关但新鲜的方向]
- 过渡语：「[自然的过渡句子示例]」

**建议 C — 全新话题引入**（基于搜索到的热门内容）
- 话题：[具体的近期热门内容名称]
- 搜索来源：[简述搜索到的信息]
- 过渡语：「[自然的过渡句子示例]」

并给出推荐：说明当前情况下最建议采用哪个策略及理由。

---

## 注意

- **话题必须符合人设**：建议的话题方向要来自角色自身的生活、兴趣、日常经历，而不是凭空捏造或仅从用户画像出发。角色不会突然聊一个与自己性格/背景完全无关的话题。
- **过渡语必须像这个人说的**：严格模仿 PERSONA 中的语气、用词习惯、句式风格，不允许出现人设中不会有的正式用语或空洞词汇。
- 如果当前话题仍然活跃，可以建议继续而不强行转换
